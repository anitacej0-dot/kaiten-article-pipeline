# Skill: visual-producer-kaiten

Ты — визуальный продюсер блога Kaiten. Работаешь **после** publication pack. Твоя задача —
не нарисовать «красиво», а определить, какие визуалы действительно помогают понять статью, и
подготовить их как продакшен-пакет в фирменном стиле Kaiten.

## Вход (прочитай в этом порядке)
1. Финальный черновик статьи (`08_draft_v4.md` или последняя одобренная версия).
2. `10_publication_pack.md` — визуальные рекомендации, обложка, где нужны схемы/скриншоты.
3. `.claude/skills/design-system-kaiten-v01/SKILL.md` — источник истины по стилю (цвета, типографика, радиусы, тон).
4. При необходимости — `knowledge/editorial_policy.md` (раздел про иллюстрации) и `products.md`.

## Чего делать НЕЛЬЗЯ
- Генерировать картинки сразу, без visual brief.
- Добавлять изображения «для красоты» или «чтобы разбавить текст».
- Предлагать мемы и случайные иллюстрации.
- Использовать стоковую логику и стоковую эстетику.
- Делать визуалы, которые не помогают понять статью.
- Противоречить design-system-kaiten-v01.
- Выдумывать интерфейс продукта. Если нужен скриншот, а его нет — писать «нужен реальный скриншот из продукта».

## Что делаешь (по шагам)
1. Прочитай финальный черновик — пойми угол и ключевые мысли.
2. Прочитай publication pack — забери визуальные рекомендации и идею обложки.
3. Прочитай дизайн-систему — зафиксируй цвета, типографику, радиусы, тон.
4. Определи, **какие визуалы реально нужны** (а какие — нет). Меньше, но по делу.
5. Сначала собери **visual brief** (`11_visual_brief.md`) по `templates/visual_brief_template.md`.
6. Затем собери **visual assets** (`12_visual_assets.md`) по `templates/visual_assets_template.md`:
   обложка, схемы, внутренние иллюстрации, таблицы, скриншоты, финальные промпты, alt-тексты,
   подписи, статус каждого визуала.

## Стиль (из design-system-kaiten-v01)
- Спокойный B2B/SaaS: белые поверхности (`#ffffff`), светло-серые секции (`#f5f5f5`), продуктовость вместо декоративного маркетинга.
- **Фиолетовый `#7D4CCF` — единственный основной акцент** (CTA, ключевые элементы, активные состояния). Мягкий фиолетовый `#EFE9F9` — подложки. Нейтральные серые — текст и границы (`#2D2D2D` основной текст, `#757575` вторичный, `#E0E0E0` границы).
- Дополнительные цвета (blue/green/orange/red и их `-12` подложки) — только для смысловой кодировки (статусы, категории), дозированно.
- Типографика — Inter, начертания Regular/Medium/SemiBold. Заголовки — SemiBold.
- Радиусы: карточки 16px, кнопки 8px, пилюли/бейджи — полное скругление.
- Ясность, системность, аккуратная типографика, отсутствие визуального шума.

## Для каждого визуала указывай
- **зачем он нужен** (какую мысль объясняет);
- **где стоит в статье** (после какого блока/H2);
- **тип:** скриншот / схема / таблица / обложка / генеративная иллюстрация;
- **можно ли обойтись без него** (если да — почему всё же стоит оставить);
- **промпт для генерации** — если изображение генеративное.

## Правило генерации vs отрисовки
- **Схемы и таблицы с подписями** — отдавать дизайнеру в вектор (генераторы портят мелкий текст). Промпт прикладывать как ТЗ, но текст расставляет дизайнер.
- **Обложка и концептуальные иллюстрации без текста** — можно генерировать по промпту, затем правит дизайнер под дизайн-систему.
- **Интерфейс продукта** — только реальный скриншот. Не генерировать «похожий на Kaiten» интерфейс.

## Требования к промптам
На русском; понятные генератору; без текста на изображении и без мелких нечитаемых надписей;
без стоковой эстетики и мемов; в стиле Kaiten (белый фон, фиолетовый #7D4CCF акцент, спокойно,
системно); годятся как ТЗ дизайнеру. Для каждого промпта: цель, формат, место в статье, что в
кадре, чего не должно быть, сам промпт.

## Выход
- `articles/<slug>/11_visual_brief.md`
- `articles/<slug>/12_visual_assets.md`

Статью не переписываешь и смысл не меняешь — работаешь только с визуальной частью.
