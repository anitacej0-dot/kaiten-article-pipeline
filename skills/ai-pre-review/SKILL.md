# Навык: ai-pre-review (AI-предредактура)

Главный удар по ручной редактуре: вернуть автору всё, что он может исправить сам, чтобы
редактор работал со смыслом, а не с запятыми. Этап 5 (см. [CLAUDE.md](../../CLAUDE.md)).

## Вход
- `articles/<папка>/04_draft.md` — черновик.
- `02_sources.md` — для проверки фактуры.
- Правила: `templates/ai_review.md` (рубрика), `styles/Kaiten/` (Vale), база знаний.

## Что делать (три слоя)
1. **Механический — Vale:** прогнать `styles/Kaiten/` по черновику. HARD (`error`) — блокируют.
2. **Смысловой — рубрика** из `templates/ai_review.md`: оценить 11 критериев (0–5), сначала рассуждение, потом балл; сверяться с `good_examples.md`, антипримеры — `bad_patterns.md`. Замечания с пометкой high/medium/low.
3. **Русские сервисы** (если доступны API/ручной прогон): Главред (вода/стоп-слова), **Тургенев (риск Баден-Бадена — обязательно)**, Орфограммка, text.ru (≥90%).
4. Self-refine: после правок прогнать повторно, пока high не закрыты.
5. Заполни отчёт по формату из `templates/ai_review.md`, сохрани в `05_ai_review.md`.

## Gate (решение)
- **Любой HARD ниже порога или Vale-error → «вернуть автору»** + список high-замечаний. К редактору НЕ уходит.
- Все HARD пройдены → ✅ к редактору; SOFT — рекомендации.

## Правила
- Замечания конкретные и actionable («что, где, как исправить»), а не «сделай лучше».
- Не переписывай за автора молча — фиксируй замечания; правки делает автор (этап 6).

## Выход
`05_ai_review.md` с решением. Дальше: автор правит (`06_author_fixed.md`) → редактор (этап 7).
