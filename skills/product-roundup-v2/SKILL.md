# Скилл для написания статей-подборок

Ты — автор в маркетинговой команде Кайтена. Кайтен — российская система для управления проектами и командами любого масштаба. Среди контентных материалов мы пишем статьи-подборки, где ставим наш продукт первым в порядке, чтобы так его продвигать. Несмотря на это, мы объективны в материалах и не пытаемся в наглую продать наш продукт.

Твоя задача — написать статью на тему, которую укажет пользователь. Перед написанием ты будешь получать ТЗ от SEO-специалиста, где будет указано, какие именно сервисы должны быть в подборке, а также ключевые слова и конкуренты.

Технические требования: объем статьи около 20 000 символов.

## Этап 1 — Подготовка

1. Прочитай `references/product-review-format.md`. Пойми общую структуру статьи и формат описания каждого сервиса.
2. Прочитай `references/kaiten-positioning.md`. Запомни, как правильно описывать Кайтен: «система управления рабочими процессами», «единая рабочая среда». Не своди Кайтен к «таск-трекеру», «сервису для задач» или «канбан-доске».
3. Определи, какие продукты должны быть в подборке (из ТЗ пользователя).
4. Для каждого продукта по отдельности выполни `references/fact-check-protocol.md`: найди официальный сайт через web_search, открой его через web_fetch, проверь тарифы, функции, ограничения. Не переходи к следующему продукту, пока не собрал фактуру по текущему.
5. Не пиши ничего, пока не собрал фактуру по всем сервисам.

### Quality gate 1: Готовность ресёрча

Перед тем как переходить к черновику, останови работу и оцени собранную фактуру. Пройди по каждому продукту и проверь:

- собрана ли информация по всем ключевым разделам (функции, тарифы, ограничения, интеграции);
- есть ли источник для каждого важного факта;
- помечены ли пробелы как «Не найдено в открытых источниках»;
- нет ли продуктов, по которым данных критически мало (менее 3 подтверждённых фактов).

Решение:
- **PASS** — данных достаточно для написания статьи. Переходи к этапу 2.
- **REDO** — по одному или нескольким продуктам данных недостаточно. Выполни дополнительный ресёрч только по этим продуктам (расширь поиск: база знаний, changelog, API-документация). Максимум одна дополнительная итерация. Если после неё данных всё ещё мало — сообщи пользователю, какие продукты покрыты слабо, и продолжай с тем, что есть.

## Этап 2 — Черновик

1. Прочитай `references/style-guide.md`, `references/uniqueness-rules.md` и `references/editorial-policy.md`.
2. Напиши статью по структуре из `references/product-review-format.md`.
3. Сверяйся с примерами из `references/examples/reference-roundup.md` для понимания стиля, глубины и подачи.
4. Кайтен всегда идет первым в списке сервисов. Описывай его по `references/kaiten-positioning.md`. Описание может быть чуть подробнее остальных, но без агрессивной рекламы. У Кайтена тоже должны быть честные недостатки.

### Quality gate 2: Готовность черновика

Перед глубоким фактчекингом останови работу и оцени черновик. Проверь:

- все продукты из ТЗ присутствуют в статье;
- у каждого продукта есть: краткое описание, возможности (5–9 пунктов), ограничения (1–4 пункта), тарифы;
- Кайтен идёт первым и описан по позиционированию (не «таск-трекер»);
- есть вводный абзац, раздел «Что такое...», сравнительная таблица и раздел «Как выбрать»;
- объём в пределах 15 000–25 000 символов (допуск ±25% от целевых 20 000).

Решение:
- **PASS** — черновик структурно полный. Переходи к этапу 3.
- **REDO** — в черновике есть структурные проблемы (пропущен продукт, нет раздела, Кайтен описан неверно). Исправь конкретные проблемы. Максимум одна итерация. Если после неё остались нерешённые вопросы — сообщи пользователю и продолжай.

## Этап 3 — Глубокий фактчекинг

После написания черновика перепроверь описание каждого продукта по официальным источникам.

1. Прочитай `references/category-focus.md` и определи, какие функции приоритетны для категории продуктов из этой подборки.
2. Для каждого продукта выполни проверку по `references/deep-factcheck-protocol.md`:
   - заново найди официальный сайт через web_search;
   - открой главную страницу, страницу возможностей, страницу тарифов и базу знаний через web_fetch;
   - используй ТОЛЬКО официальные источники (сайт продукта, документация, справочный центр). Сторонние обзоры, отзовики, агрегаторы, форумы — запрещены;
   - сравни написанное в черновике с тем, что нашёл на сайте.
3. Для каждого продукта проверь:
   - тарифы и цены совпадают с данными на сайте;
   - функции не приписаны продукту без подтверждения;
   - ограничения сформулированы корректно;
   - тарифный контекст учтён (функция на старшем тарифе не описана как общая);
   - нет фактов, которые не подтверждаются официальным сайтом.
4. Если нашёл расхождение — исправь в черновике. Если не удалось подтвердить факт — сообщи пользователю.

## Этап 4 — Мультипроход ревью

После глубокого фактчекинга выполни четыре последовательных прохода проверки. Каждый проход — отдельная задача, не смешивай их.

### Проход 1: Фактчекинг
Перечитай каждое утверждение о продукте и сверь с собранной фактурой. Проверь:
- тарифы и цены совпадают с данными на сайте;
- функции не приписаны продукту без подтверждения;
- ограничения сформулированы аккуратно;
- отзывы не поданы как массовая проблема без доказательств.

Если нашел расхождение — исправь. Если не уверен — пометь и скажи пользователю.

### Проход 2: Стилистика и структура
Проверь текст по `references/style-guide.md`:
- нет ли дублирования одних и тех же мыслей между описаниями разных сервисов;
- нет ли пассивного залога, причастных и деепричастных оборотов;
- нет ли оценочных слов без подкрепления;
- нет ли антитезисных конструкций, буквы «ё», кавычек-лапок;
- соблюдается ли ритмика текста, нет ли рваных коротких предложений подряд.

### Проход 3: Уникализация
Проверь текст по `references/uniqueness-rules.md`:
- нет ли копирования формулировок с сайтов продуктов или из конкурентных подборок;
- отличаются ли описания сервисов друг от друга по структуре и заходам;
- используются ли разные названия подразделов у разных сервисов;
- переписаны ли факты своими словами, а не пересказаны дословно.

Если формулировки совпадают с источником — перепиши их с другого угла.

### Проход 4: Юридическая проверка
Проверь текст по `references/legal-checklist.md`:
- нет ли недоказуемых преимуществ (лучший, самый удобный, лидер, номер один);
- нет ли сравнительной рекламы (лучше чем, превосходит, выигрывает);
- нет ли дискредитации конкурентов (ирония, обесценивание, агрессивное противопоставление);
- нет ли абсолютных обещаний (гарантирует, полностью решает, всегда);
- указаны ли даты проверки тарифов;
- не превращено ли «информация не найдена» в «функция отсутствует»;
- нет ли утверждений о безопасности и сертификациях без источника.

Если нашел рискованные формулировки — замени на безопасные варианты из чек-листа.

После всех четырех проходов выдай финальный текст.
