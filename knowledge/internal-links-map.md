# Карта внутренней перелинковки

> Ключевые страницы и правила перелинковки — чтобы каждая статья вела на релевантные
> страницы блога и продукта с осмысленным anchor-текстом. Используется в `build-outline`
> и `package-for-cms`. Связано с [[seo_policy]], [[target-keywords]].
>
> ⚠️ Засеяно известными ключевыми страницами. **Полный список статей блога** команда
> добавляет через поиск по kaiten.ru/blog (как и предписывает редполитика).

## Правила перелинковки (из редполитики/SEO)
- Обязательно добавляем ссылки на другие статьи блога Kaiten (через поиск по блогу).
- **Anchor-текст осмысленный**: не «здесь»/«читать», а по теме целевой страницы.
- Связываем статьи внутри одного кластера ([[target-keywords]]) — это сигнал экспертизы.
- Не переспамливать ссылками (риск Баден-Бадена, см. [[seo_policy]]).

## Ключевые целевые страницы (хабы)

| Страница | URL | Когда ставить ссылку |
| --- | --- | --- |
| Блог Kaiten | kaiten.ru/blog | общий хаб |
| Кейсы клиентов | kaiten.ru/blog/tag/case | при упоминании «было → стало», social proof |
| Service Desk | kaiten.ru/features/service-desk | темы про заявки, поддержку, ITSM |
| Тарифы | kaiten.ru (страница цен) | при упоминании Free/Pro/Enterprise |
| Возможности продукта | kaiten.ru/features/... | при разборе функций (Ганта, канбан, автоматизации) |

## Типовые связки (примеры)
- Статья «Как построить Service Desk» → страница Service Desk + кейсы с Service Desk (АТИ.SU, ЦНИИ Эпидемиологии, ActiveCloud).
- Статья «Аналоги/замена Jira» → страница миграции + кейсы миграции (X5, АльфаСтрахование, Додо Пицца).
- Экспертная статья про канбан/scrum → продуктовые страницы канбан-досок / спринтов.
- Отраслевая статья → релевантный кейс того же сегмента из [[good_examples]].
