# SEO-правила Kaiten

> Требования к оптимизации статей блога. Используется при сборке брифа ([[../templates/article_brief]])
> и публикационного пакета ([[../templates/cms_package]]). Извлечено из реальных ТЗ
> (Admin24, Service Desk). Структурные требования согласованы с [[editorial_policy]].

## Мета-теги и URL

- **Title** — содержит ключ, отражает суть, год для актуальности. Примеры: «Аналоги Admin24: замена и альтернатива Admin24 в 2026 году»; «Как построить Service Desk: этапы, SLA, роли и примеры процессов».
- **Description** — краткое описание с ключами и пользой. Пример: «Разбираем, как построить Service Desk: с чего начать, как описать заявки, настроить SLA, распределить роли и избежать ошибок внедрения».
- **Post URL** — короткий слаг латиницей, по ключу: `analogi-admin24`, `kak-postroit-service-desk`.

## Работа с ключами

- Ключевые фразы **равномерно распределяем** по тексту.
- Каждый ключ — минимум один раз **в любой словоформе**.
- По возможности используем ключи **в подзаголовках** (H2/H3).
- **Первое предложение под H1** может содержать побуждение к действию — оно может выводиться в сниппет поисковика.

## Структура под поиск и под нейросети (GEO/AEO)

Статья должна быть удобна и людям, и нейросетям, которые извлекают из неё ответы:

- В начале — **короткое определение в 2–3 предложениях** (что это).
- Чёткая предсказуемая структура: определение → когда нужно → из чего состоит → как сделать → как адаптировать → ошибки → метрики → что запомнить.
- **Списки, таблицы, чек-листы, пошаговые инструкции** — обязательно.
- **Самостоятельные смысловые блоки**, которые можно процитировать отдельно.
- **Блок FAQ** в конце.
- Без длинных вступлений без ответа. Без абстракций «повышает эффективность / оптимизирует процессы» без объяснения, как именно.

## Заголовки H1

Предлагаем 2–3 варианта H1. Базовый принцип — ключ + конкретная польза/охват. Пример для
темы Service Desk: «Как построить Service Desk в компании: пошаговый план для ИТ, HR, АХО
и поддержки».

## Сравнительные статьи и подборки

- Изучаем конкурентов в выдаче (SERP) по теме — собираем в research.
- Для подборок сервисов: краткое описание → преимущества → недостатки → цена → сравнительная таблица → «как выбрать». Объём описания сервиса — ~1200–1500 знаков.
- Плашка «это не рейтинг» обязательна (см. правила про конкурентов в [[editorial_policy]]).

## Перелинковка и проверки

- Обязательно добавляем ссылки на другие статьи блога Kaiten (через поиск по блогу).
- Акценты выносим отдельным блоком.
- Проверка оригинальности на text.ru — **от 90%**, прикладываем скрин.
- Иллюстрации: скриншоты, инфографика, схемы, таблицы, мемы в тему — без стоков и «лишь бы было» (детали — в [[editorial_policy]]).

## Объём

Зависит от типа статьи. Ориентиры из практики: подборка сервисов — по объёму описаний;
большая практическая статья — ~15 000 знаков с пробелами.

## Русская выдача: Яндекс, нейроответы, Баден-Баден

Специфика РФ (подробно — [research/04](../research/04_russian_specifics.md)).

### ⚠️ Фильтр «Баден-Баден» — баланс ключей
Яндекс штрафует **переоптимизированные тексты** (переспам ключей, вода). Правило
«равномерно распределить ключи» работает только вместе с этим ограничением: распределяем —
да, но **без передозировки**. Естественность важнее плотности. Проверка — сервис
**Тургенев** (обязательный гейт для SEO-статей).

### Формат под нейроответы Яндекса и Алисы
Нейроответы — уже большая доля выдачи; AI выбирает всего 2–3 источника для цитаты. Чтобы
попасть в цитату:
- страница в **топ-30** обычной выдачи;
- **короткое резюме/определение в начале** (2–3 предложения);
- полные самодостаточные ответы; заголовки **H2–H4**; списки; **HTML-таблицы** для сравнений;
- **короткие FAQ-блоки** — почти главный источник цитирования;
- **микроразметка** (schema: Article, FAQ).

### Приоритеты ранжирования (2025–2026)
**Поведение → польза → доверие → релевантность.** Антиспам против генеративного контента —
ещё один довод за «человек по краям» и против нейрослопа. Консистентность фактов о Kaiten
на сайте и во внешних источниках повышает доверие и цитируемость ([[products]]).

### GEO-усилители (Princeton GEO study)
Повышают цитируемость нейросетями: **экспертные цитаты (~+41%), статистика (~+30%),
ссылки на источники (~+30%)**. Закладываем в бриф и проверяем в AI-ревью.
