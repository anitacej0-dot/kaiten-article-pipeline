## 1. Читаемость русского текста

Индекс Флеша создан для английского. Для русского есть **адаптация Оборневой**: та же
формула, но с коэффициентами под русскую морфологию (в среднем слова длиннее и слогов больше):

```
УФ ≈ 206.835 − (1.015 × средняя длина предложения в словах)
              − (84.6 × среднее число слогов на слово)
```
(коэффициенты в адаптациях варьируются — Оборнева, Сиротина и др.)

**Что берём нам:**
- Точную русскую читаемость считают **Орфограммка** и спец-сервисы (есть индекс Флеша для русского).
- Для конвейера достаточно **прокси-метрики**, которая уже в нашей редполитике: **средняя длина предложения ≤15 слов**. Её Vale считает напрямую (`metric`, формула `words/sentences`) — язык не важен. Это наш стартовый автопоказатель читаемости.
- Дробный индекс Флеша-Оборневой — опционально, как второй сигнал (через внешний сервис).

## 2. Русские сервисы авторедактуры (уже в нашем процессе)

В редполитике уже прописаны Орфограммка, Главред, text.ru. Их нужно **встроить в конвейер
как автоматические проверки на конкретных этапах**, а не «помнить, что надо прогнать».

| Сервис | Что проверяет | Где в конвейере | API |
| --- | --- | --- | --- |
| **Главред** (glvrd.ru) | стоп-слова, вода, модальность, пассив, канцелярит, оценки — **прямо наша редполитика** | AI-предредактура (этап 6) | ограниченно |
| **Тургенев** (Ашманов) | риск фильтра «Баден-Баден», переспам, повторы, читаемость, стилистика | SEO-проверка перед сдачей | **есть API** |
| **Орфограммка** | орфография, пунктуация, грамматика + индекс Флеша | финальная вычитка | есть API |
| **text.ru** | уникальность (цель ≥90%), вода, заспамленность | финал, перед публикацией | есть |

→ **Главред почти дублирует наш `forbidden_phrases` и редполитику** — то есть наш чёрный
список и Vale-правила идут в ту же сторону, что отраслевой стандарт. Тургенев закрывает
специфический русский риск (см. ниже).

## 3. Vale для русского — что работает, что нет

| Тип проверки | Русский | Вывод |
| --- | --- | --- |
| existence, substitution, occurrence, consistency (регулярки) | ✅ работает сразу | весь `forbidden_phrases.md` ложится без доработок |
| metric (читаемость по своей формуле) | ✅ (длина предложений) | стартовая метрика читаемости |
| spelling | ⚠️ нужен Hunspell `ru_RU.dic` + `ru_RU.aff` в `<StylesPath>/config/dictionaries` | можно, но орфографию лучше отдать Орфограммке |
| sequence (части речи), англ. readability-формулы | ❌ нужны русские NLP-модели | не используем |

**Готовых русских style-паков для Vale нет** — мы пишем свой `styles/Kaiten/` (черновик
правил уже в [research 02](02_deep_dive_top_solutions.md)). Это нормально: суть Vale —
кодифицировать СВОЮ редполитику, а не брать чужую.

## 4. Яндекс-выдача и нейроответы (GEO под РФ)

Главная специфика, которой нет в англоязычных гайдах.

### Фильтр «Баден-Баден» — важное ограничение
Яндекс штрафует переоптимизированные SEO-тексты (переспам ключей, вода). **Это в прямом
напряжении с нашим правилом «равномерно распределить ключи»** из ТЗ: распределять — да,
но без передозировки. **Тургенев проверяет именно риск Баден-Бадена** → ставим его как
обязательный gate для SEO-статей. Совпадает с редполитикой (краткость, польза, без воды).

### Как попасть в нейроответы Яндекса / Алисы
Нейроответы — уже ~68% информационных блоков выдачи (конец 2025). Чтобы попасть в цитату:
- страница в **топ-30** обычной выдачи;
- **короткое резюме/определение в начале** (2–3 предложения) — совпадает с нашим `seo_policy`;
- полные самодостаточные ответы; заголовки **H2–H4**; списки; **HTML-таблицы** для сравнений; **микроразметка** (schema);
- **короткие FAQ-блоки — почти главный источник цитирования**.

AI выбирает всего **2–3 источника** для цитаты — конкуренция жёстче, чем за топ-10 ссылок.

### Приоритеты ранжирования 2025–2026
**Поведение → польза → доверие → релевантность.** Алгоритмы: YATI (семантика),
Королёв (поведение, long-tail), Vega (интент). Антиспам против генеративного контента —
ещё один довод за «человек по краям» и против нейрослопа.

### Русские нейросети как GEO-цели
YandexGPT / Нейро, Алиса, GigaChat (Сбер). Для них работают те же принципы извлекаемости
+ консистентность фактов о Kaiten на сайте и во внешних источниках ([[products]]).

## Вывод
Русский язык не ломает наш план, а уточняет его: **регулярочные проверки Vale и наш
чёрный список работают для русского сразу**; орфография/читаемость — на русские сервисы
(Орфограммка), которые у команды уже есть; Главред идейно совпадает с нашей редполитикой;
а главное русское ограничение — **Баден-Баден** (не переспамить ключи) и **формат под
нейроответы Яндекса** (резюме в начале + FAQ + таблицы) — добавляем в `seo_policy.md`.
